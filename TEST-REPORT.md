## 三、发现的Bug汇总

### Bug #1：存储写入失败（saveRecordsToStorage return false）
- **严重度**：P0 / Blocker
- **影响范围**：所有写入操作（新增、编辑保存、删除）
- **现象**：
  - 新增记录：表单不重置、无成功/错误提示、记录未保存
  - 编辑保存：Dialog不关闭、修改未持久化
  - 删除：确认后记录不消失
- **根因**：`index.html` L494 `saveRecordsToStorage()` 函数 return false
- **受影响用例**：L1-01, L1-02, L1-05, L2-04, L3-06, L3-07(BLOCKED)

### Bug #2：收入/支出切换分类不联动
- **严重度**：P1 / Major
- **现象**：切换类型为"收入"后，分类下拉仍显示8个支出分类(餐饮/交通/购物/居住/娱乐/医疗/教育/其他)，未切换为4个收入分类(工资/奖金/投资/其他)
- **根因**：`index.html` `getCategoriesByType(type)` 函数硬编码 `return EXPENSE_CATEGORIES`，忽略 `type` 参数。正确应为 `return type === 'income' ? INCOME_CATEGORIES : EXPENSE_CATEGORIES`
- **受影响用例**：L1-02, L2-02

### Bug #3：正数结余颜色不是绿色
- **严重度**：P2 / Minor
- **现象**：结余为正数(¥2,000.00)时，金额文字未显示为绿色(预期 #10B981)
- **受影响用例**：L1-03, L2-11a

### Bug #4：删除逻辑条件反转（idx !== -1 时 return）
- **严重度**：P0 / Blocker
- **影响范围**：删除操作
- **现象**：点击确认删除后记录不消失
- **根因**：`index.html` L724 `if (idx !== -1) return` — 条件反转，找到目标记录时直接 return，splice 永远不执行。正确应为 `if (idx === -1) return`
- **与 Bug #1 的关系**：独立缺陷。即使修复 Bug #1，Bug #4 不修则删除仍不工作
- **受影响用例**：L1-05, L3-06

### Bug #5：超预算进度条颜色未变红
- **严重度**：P2 / Minor
- **现象**：支出(¥6,000)超过预算(¥5,000)时，进度条填充色为灰色(#E8DEFF)，而非预期的红色。`progressColor` 属性硬编码
- **受影响用例**：L2-08, L3-04

### 疑似问题：超预算 Notification 未弹出
- **严重度**：待确认
- **现象**：L2-08 预期页面加载后自动弹出 Notification"本月支出已超预算!"，实际未弹出任何通知
- **可能原因**：Notification 逻辑依赖 saveRecordsToStorage 成功后触发(被 Bug #1 阻塞)，或 Notification 组件本身未实现
- **需修复 Bug #1 后重测确认**

## 六、结论

1. **P0 阻塞缺陷（2个）**：Bug #1 (存储写入失败) 导致所有写操作失效；Bug #4 (删除条件反转) 额外阻塞删除功能
2. **P1 功能缺陷（1个）**：Bug #2 (分类联动失效) 导致收入记录无法使用正确分类
3. **P2 视觉缺陷（2个）**：Bug #3 (正数结余非绿色)、Bug #5 (超预算进度条非红色)
4. **只读功能基本正常**：统计看板数值、分类占比、月度趋势、日期分组小计、金额校验、备注上限均工作正确
5. **待确认**：超预算 Notification 弹出功能需修复 Bug #1 后重测

**建议**：优先修复 Bug #1 + Bug #4，然后修复 Bug #2，最后修复 Bug #3 + Bug #5。修复后需重测（共 10 条）：
- L1-01, L1-02, L1-05, L2-04, L3-06, L3-07 — Bug #1/#4 修复后重测写入功能
- L1-03 — Bug #3 修复后重验结余颜色
- L2-02 — Bug #2 修复后重验分类联动
- L2-08 — Bug #5 修复后重验进度条颜色 + Notification
- L1-04 — **需更新 fixture 日期**（非代码 bug，fixture 中日期 4/20-4/22 已过期，需改为相对于执行日的 today/yesterday 日期，或在 Pre-flight 中动态设置）
