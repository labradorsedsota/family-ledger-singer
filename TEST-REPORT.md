## 三、发现的Bug汇总

### Bug #1：存储写入失败（saveRecordsToStorage return false）
- **严重度**：P0 / Blocker
- **影响范围**：所有写入操作（新增、编辑保存、删除）
- **现象**：
  - 新增记录：表单不重置、无成功/错误提示、记录未保存
  - 编辑保存：Dialog不关闭、修改未持久化
  - 删除：确认后记录不消失
- **根因**：`app.js` L494 `saveRecordsToStorage()` 函数 return false
- **受影响用例**：L1-01, L1-02, L1-05, L2-02, L2-03, L2-04, L3-06

### Bug #2：收入/支出切换分类不联动（疑似）
- **严重度**：P1 / Major
- **现象**：切换类型为"收入"后，分类下拉仍显示支出分类(餐饮/交通/购物等)，未切换为收入分类(工资/奖金/投资/其他)
- **受影响用例**：L1-02, L3-07

### Bug #3：正数结余颜色不是绿色
- **严重度**：P2 / Minor
- **现象**：结余为正数(¥2,000.00)时，金额文字未显示为绿色。CSS class `.amount-income` 可能存在颜色值错误
- **受影响用例**：L2-11a

### Bug #4：删除逻辑条件反转（idx !== -1 时 return）
- **严重度**：P0 / Blocker
- **影响范围**：删除操作
- **现象**：点击确认删除后记录不消失（与 Bug #1 共同阻塞删除功能）
- **根因**：`index.html` L724 `if (idx !== -1) return` — 条件反转，找到目标记录时直接 return，splice 永远不执行。正确应为 `if (idx === -1) return`
- **与 Bug #1 的关系**：独立缺陷。即使修复 Bug #1，Bug #4 不修则删除仍不工作
- **受影响用例**：L1-05, L3-06

### Bug #5：超预算进度条颜色未变红
- **严重度**：P2 / Minor
- **现象**：支出(¥6,000)超过预算(¥5,000)时，进度条填充色为灰色/淡紫色(#E8DEFF)，而非预期的红色。`progressColor` 属性硬编码
- **受影响用例**：L3-04

## 五、结论

1. **核心阻塞**：Bug #1 (saveRecordsToStorage return false) 导致所有写入操作失效；Bug #4 (删除条件反转) 额外阻塞删除功能。两者均为P0级
2. **只读功能正常**：统计看板、分类占比、月度趋势、日期分组等展示型功能均工作正常
3. **校验逻辑正常**：金额空值校验、编辑清空校验、备注50字上限均正确
4. **视觉细节**：2个CSS颜色问题（正数结余非绿色、超预算进度条非红色）

**建议**：优先修复 Bug #1 + Bug #4 后重测所有写入类用例（L1-01, L1-02, L1-05, L2-02~L2-04, L3-06, L3-07）。
