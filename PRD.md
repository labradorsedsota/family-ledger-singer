## 第 2 章 产品目标与背景

### 2.1 背景

用户需要一个简单的工具来记录日常家庭收支，了解钱花在哪里、每月花了多少。市面记账 App 功能臃肿、需要注册登录，用户只想打开浏览器快速记一笔。

### 2.2 产品目标

提供一个**纯前端、打开即用**的家庭记账工具：
- 快速记录每笔收入/支出
- 按分类汇总，直观了解消费结构
- 月度统计与趋势查看
- 月预算管控，超支及时提醒

### 2.3 成功标准

- 用户能在 10 秒内完成一笔记账
- 统计看板一眼看清本月收支概况
- 数据持久化在浏览器本地，刷新不丢失

### 2.4 不做什么

- ❌ 后端 / 数据库 / 用户登录
- ❌ 多用户 / 多账户
- ❌ 数据导出（CSV/Excel）
- ❌ 移动端适配
- ❌ 多币种
- ❌ 分类预算（只做月总预算）

## 第 4 章 功能需求

### 4.1 记账表单（F-01）

用户通过表单快速录入一笔收支记录。

**字段：**

| 字段 | 组件 | 必填 | 说明 |
|------|------|------|------|
| 类型 | Radio | 是 | 支出（默认）/ 收入 |
| 金额 | InputNumber | 是 | 正数，最多 2 位小数，最大 9,999,999.99 |
| 分类 | Select | 是 | 支出 8 类 / 收入 4 类（见 5.1） |
| 日期 | DatePicker | 是 | 默认今天，不可选未来日期 |
| 备注 | Input | 否 | 最多 50 字符 |

**AC：**

- **AC-01-01** Given 用户打开应用，When 页面加载完成，Then 记账表单显示在页面顶部区域，类型默认选中"支出"，日期默认今天
- **AC-01-02** Given 用户填写了金额、选择了分类，When 点击"记一笔"按钮，Then 记录保存到 localStorage，表单清空，显示成功提示（Message），账单列表和统计卡片实时刷新
- **AC-01-03** Given 用户未填写金额或未选分类，When 点击"记一笔"，Then 对应字段显示校验错误提示，不提交
- **AC-01-04** Given 用户输入金额为 0 或负数，When 点击"记一笔"，Then 金额字段显示校验错误"金额必须大于 0"
- **AC-01-05** Given 用户输入金额超过 9,999,999.99，When 失去焦点，Then 金额字段显示校验错误"金额不能超过 9,999,999.99"
- **AC-01-06** Given 用户切换类型为"收入"，When 切换完成，Then 分类下拉选项切换为收入分类列表

### 4.2 账单列表（F-02）

按日期分组展示所有账单记录。

**展示方式：** Collapse 折叠面板，按日期降序分组（最近的在上面），每组标题显示日期和当日小计。

**每条记录显示：** 分类图标 + 分类名 | 备注 | 金额（支出红色，收入绿色）

**操作：** 每条记录支持编辑和删除。

**AC：**

- **AC-02-01** Given 存在账单记录，When 页面加载，Then 账单按日期降序分组展示，默认展开今天和昨天的分组
- **AC-02-02** Given 某日有 3 笔支出共 150 元、1 笔收入 50 元，When 查看该日分组标题，Then 显示"4月22日（周二）支出 ¥150.00 收入 ¥50.00"
- **AC-02-03** Given 用户点击某条记录的编辑按钮，When 编辑弹窗打开，Then 表单预填该记录所有字段，可修改后保存
- **AC-02-04** Given 用户点击某条记录的删除按钮，When 确认删除（MessageBox），Then 记录从 localStorage 移除，列表和统计实时刷新
- **AC-02-05** Given 没有任何账单记录，When 页面加载，Then 显示 Empty 空状态"还没有记账记录，开始记第一笔吧"

### 4.3 统计看板（F-03）

Bento Grid 布局的统计卡片区域，展示本月核心数据。

**卡片内容：**

| 卡片 | 数据 | 组件 |
|------|------|------|
| 本月支出 | 当月支出总额 | Card + 大字号金额 |
| 本月收入 | 当月收入总额 | Card + 大字号金额 |
| 本月结余 | 收入 - 支出 | Card + 大字号金额（正绿负红） |
| 预算进度 | 已用 / 月预算 | Card + Progress 进度条 |

**AC：**

- **AC-03-01** Given 本月有支出记录，When 查看统计看板，Then "本月支出"卡片显示所有支出之和，格式 ¥X,XXX.XX
- **AC-03-02** Given 本月收入 5000、支出 3000，When 查看"本月结余"卡片，Then 显示 ¥2,000.00，绿色
- **AC-03-03** Given 本月收入 3000、支出 5000，When 查看"本月结余"卡片，Then 显示 -¥2,000.00，红色
- **AC-03-04** Given 月预算 5000、本月支出 3500，When 查看"预算进度"卡片，Then 进度条显示 70%，文字"已用 ¥3,500.00 / ¥5,000.00"
- **AC-03-05** Given 本月支出超过月预算，When 超支发生时，Then 进度条变红，触发 Notification 提醒"本月支出已超预算！"
- **AC-03-06** Given 当月没有任何记录，When 查看统计看板，Then 所有金额显示 ¥0.00，预算进度 0%

### 4.4 分类支出占比（F-04）

以可视化方式展示本月各支出分类的占比。

**展示方式：** 横向条形图或分类列表，每个分类一行：分类名 + 金额 + 占比百分比 + Progress 进度条。按金额降序排列。

**AC：**

- **AC-04-01** Given 本月餐饮支出 2000、交通 500、购物 1500，When 查看分类占比，Then 餐饮显示 50%、购物 37.5%、交通 12.5%，按金额降序排列
- **AC-04-02** Given 本月没有支出，When 查看分类占比，Then 显示 Empty 空状态
- **AC-04-03** Given 某分类支出为 0，When 查看分类占比，Then 该分类不显示在列表中

### 4.5 月预算设置（F-05）

用户可设置每月预算额度。

**入口：** 统计看板的预算卡片上有"设置"按钮，点击弹出 Dialog。

**字段：** 月预算金额（InputNumber），最小 0，最大 9,999,999.99，2 位小数。

**AC：**

- **AC-05-01** Given 用户从未设置预算，When 查看预算卡片，Then 显示"未设置预算，点击设置"
- **AC-05-02** Given 用户点击设置，When Dialog 打开，Then 如已有预算值则预填，否则为空
- **AC-05-03** Given 用户输入 5000 并确认，When 保存成功，Then 预算值存入 localStorage，预算进度卡片立即刷新，显示 Message 提示"预算设置成功"
- **AC-05-04** Given 用户将预算设为 0，When 确认保存，Then 等同于清除预算，预算卡片回到"未设置预算"状态

### 4.6 月度收支趋势（F-06）

展示近 6 个月的收支趋势。

**展示方式：** Timeline 组件，每个月一个节点，显示该月收入、支出、结余。

**AC：**

- **AC-06-01** Given 有 3 个月的数据，When 查看趋势，Then 显示 3 个月的节点，按时间正序排列（最早在上）
- **AC-06-02** Given 某月没有记录，When 查看趋势，Then 该月节点显示收入 ¥0.00、支出 ¥0.00、结余 ¥0.00
- **AC-06-03** Given 只有当月数据，When 查看趋势，Then 只显示当月 1 个节点

### 4.7 编辑账单（F-07）

编辑已有记录。

**入口：** 账单列表中每条记录的编辑图标按钮。

**方式：** Dialog 弹窗，表单与记账表单一致，预填当前值。

**AC：**

- **AC-07-01** Given 用户点击编辑某条支出记录，When Dialog 打开，Then 所有字段预填该记录值（类型、金额、分类、日期、备注）
- **AC-07-02** Given 用户修改金额后保存，When 保存成功，Then localStorage 更新，账单列表和统计看板实时刷新，显示 Message "修改成功"
- **AC-07-03** Given 用户在编辑弹窗中清空金额，When 点击保存，Then 显示校验错误，不允许保存

## 第 5 章 业务规则与边界条件

### 5.1 收支分类

**支出分类（8 类）：**

| 编码 | 名称 | 图标建议 |
|------|------|---------|
| food | 餐饮 | 🍽️ |
| transport | 交通 | 🚗 |
| shopping | 购物 | 🛒 |
| housing | 居住 | 🏠 |
| entertainment | 娱乐 | 🎮 |
| medical | 医疗 | 🏥 |
| education | 教育 | 📚 |
| other_expense | 其他 | 📦 |

**收入分类（4 类）：**

| 编码 | 名称 | 图标建议 |
|------|------|---------|
| salary | 工资 | 💰 |
| bonus | 奖金 | 🎁 |
| investment | 投资 | 📈 |
| other_income | 其他 | 💵 |

分类为预设固定值，不支持用户自定义。

### 5.2 金额计算规则

- **精度：** 所有金额存储和计算使用 2 位小数（分精度）
- **舍入规则：** 四舍五入（`toFixed(2)`）
- **汇总计算：** 先累加原始值，最后一步 toFixed(2)，避免中间舍入累积误差
- **显示格式：** ¥X,XXX.XX（千分位逗号分隔）
- **金额范围：** 0.01 ~ 9,999,999.99

### 5.3 数据存储

- **存储方式：** localStorage
- **Key：** `feb_records`（账单记录数组）、`feb_budget`（月预算金额）
- **单条记录结构：**
  ```json
  {
    "id": "uuid-v4",
    "type": "expense" | "income",
    "amount": 25.50,
    "category": "food",
    "date": "2026-04-22",
    "note": "午餐",
    "createdAt": 1776835200000,
    "updatedAt": 1776835200000
  }
  ```
- **localStorage 容量：** 约 5MB，预估支持 ~50,000 条记录

### 5.4 月度计算规则

- "本月"定义：当前自然月（1 日 00:00 ~ 月末 23:59），基于记录的 `date` 字段
- 月度统计只计算 `date` 在该月范围内的记录
- 趋势展示最近 6 个自然月（含当月）

### 5.5 预算规则

- 预算为月总预算，不区分分类
- 只比对支出总额，收入不参与预算计算
- 预算进度 = 本月支出 / 月预算 × 100%
- 超支判定：本月支出 > 月预算
- 超支时进度条变红 + Notification 提醒（每次打开页面最多提醒一次，不重复弹出）

## 第 6 章 异常与错误设计

### 6.1 表单校验错误

| 场景 | 错误提示 |
|------|---------|
| 金额为空 | "请输入金额" |
| 金额 ≤ 0 | "金额必须大于 0" |
| 金额 > 9,999,999.99 | "金额不能超过 9,999,999.99" |
| 未选分类 | "请选择分类" |
| 备注超过 50 字 | "备注最多 50 个字符" |

### 6.2 数据异常

| 场景 | 处理方式 |
|------|---------|
| localStorage 读取失败 | 显示 Alert 提示"数据读取失败，请刷新页面"，页面可用但数据为空 |
| localStorage 写入失败（空间满） | 显示 Notification 错误提示"存储空间已满，请清理旧数据后重试"，本次操作不生效 |
| 数据格式损坏 | 尝试解析，跳过损坏记录，显示 Alert 警告"部分数据异常已跳过" |

### 6.3 边界场景

| 场景 | 预期行为 |
|------|---------|
| 同一秒内快速多次点击"记一笔" | 提交后按钮 disable 1 秒，防止重复提交 |
| 删除最后一条记录 | 列表显示 Empty 空状态，统计归零 |
| 跨月查看（月初切换） | 统计看板跟随系统日期自动切换到新月 |

- 统计看板区使用 CSS Grid，卡片大小不等（大卡片 2:1、小卡片 1:1）

- 卡片有圆角（12px）和微阴影
- 操作区左右比例约 1:2（表单窄，列表宽）
- 页面最大宽度 1200px，居中显示
- 最小宽度 900px（桌面端，不做响应式）

### 7.3 Header

- 产品名称"家庭记账本"居中
- 右侧：当前月份显示 + 左右箭头切换月份（可查看历史月份的统计和账单）
- 切换月份时，统计看板和账单列表同步切换到对应月份

## 第 8 章 视觉风格与设计规范

> ⚠️ v1.1 变更：视觉风格从 Apple 风简约切换为 Neo-brutalism，布局结构不变。

### 8.1 整体风格

**Neo-brutalism（新残酷主义）** — Brutalist Web Design 风格：
- 粗黑边框，实体硬阴影（无模糊）
- 高对比撞色（黄、粉、薄荷绿）
- 粗体大字排版，视觉冲击力强
- 复古现代感，卡片棱角分明
- 保留 Bento Grid 模块化布局结构

### 8.2 配色方案

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色 | `#000000` | 黑色，边框/阴影/文字 |
| 强调色-黄 | `#FFE156` | 亮黄，主按钮背景/高亮卡片 |
| 强调色-粉 | `#FF6B9D` | 粉色，支出金额/超预算/警告 |
| 强调色-绿 | `#88D9A8` | 薄荷绿，收入金额/正结余 |
| 强调色-蓝 | `#7EC8E3` | 天蓝，进度条/链接 |
| 强调色-紫 | `#C3A6FF` | 淡紫，预算卡片背景 |
| 背景色 | `#FFFDF7` | 微暖白，页面背景 |
| 卡片背景 | `#FFFFFF` | 白色（不同卡片可用不同强调色做浅底） |
| 文字主色 | `#000000` | 纯黑 |
| 文字次色 | `#333333` | 深灰 |
| 边框色 | `#000000` | 纯黑，2-3px |

### 8.3 字体

- 主字体：`'Space Grotesk', 'Inter', system-ui, sans-serif`（几何感粗体无衬线）
- 金额数字：`'Space Mono', 'JetBrains Mono', monospace`（等宽粗体）
- 标题：28-32px，font-weight 800（extra-bold）
- 金额大小：统计卡片 36px bold，列表项 16px bold
- 正文：14-16px，font-weight 500-600

### 8.4 组件样式

| 组件 | 样式规范 |
|------|---------|
| Card | 圆角 0-4px（近直角），边框 `2px solid #000`，硬阴影 `4px 4px 0 #000`，内边距 20px |
| Button 主按钮 | 背景 #FFE156，黑色文字 bold，边框 `2px solid #000`，硬阴影 `3px 3px 0 #000`，圆角 0-4px，高度 44px，hover 时阴影消失 + 位移（按压感） |

- 卡片内边距：20px

### 8.6 Neo-brutalism 设计原则

1. **所有可见元素都有黑色边框** — 卡片、按钮、输入框、进度条均有 2-3px 黑色实线边框
2. **硬阴影无模糊** — 阴影统一用 `Xpx Xpx 0 #000`，不用 blur
3. **交互反馈用阴影位移** — hover/active 时阴影减小 + 元素位移，模拟物理按压
4. **每张统计卡片可有不同底色** — 支出卡片浅粉底、收入卡片浅绿底、结余卡片浅黄底、预算卡片浅紫底
5. **文字全部偏粗** — 最细 font-weight 500，标题 800
